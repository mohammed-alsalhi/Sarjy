Sarjy Product Requirements Document · v1.0

Overview

Sarjy is a production-grade AI voice assistant with persistent memory, real-time tool execution, multilingual support, a talking avatar, and telephony — callable from any phone.

"It should feel like talking to someone who actually knows you — and gets smarter every conversation."

1 Problem Statement

Most existing voice assistants are stateless — they forget everything between sessions, don't support natural conversation memory, and can't be extended with custom tools or workflows. Sarjy solves all three.
🧠

No Memory

Current assistants start fresh every session. Users repeat themselves constantly.

⏱️

High Latency

Most voice pipelines take 3–5s end-to-end. Sarjy delivers first audio in ~1.2s.

🔒

No Extensibility

Consumer assistants can't be given custom tools, workflows, or integrations.

📱

Platform Lock-in

Tied to specific devices. Sarjy is web-first, installable as a PWA, and callable via any phone.

Memory Architecture

Short-term — Upstash Redis

Last 20 messages per user. 24-hour TTL. Retrieved on every request to give Claude conversation context. Sub-5ms reads from Vercel edge.

Long-term — Supabase Postgres

Persistent facts: name, preferences, todos, reminders. Written by Claude via save_memory tool. Row-level security ensures per-user isolation.

Telephony (Twilio) Real phone number. Inbound call → TwiML Gather STT → Claude Haiku → Polly TTS → looped conversation. P2 Done
